Screen Command in Linux with Examples

Last Updated: 12 Jul, 2024

The screen command in Linux provides the ability to launch and use multiple shell sessions from a single ssh session. When a process is started with ‘screen’, the process can be detached from the session and then can reattach the session at a later time. When the session is detached, the process that was originally started from the screen is still running and managed by the screen itself. The process can then re-attach the session at a later time, and the terminals are still there, the way it was left.

Syntax:

screen [-opts] [cmd [args]]

Options:

  • -a: It forces all capabilities into each window’s termcap.
  • -A -[r|R]: It adapts all windows to the new display width & height.
  • -c file: It reads configuration file instead of ‘.screenrc’.
  • -d (-r): It detaches the elsewhere running screen (and reattach here).
  • -dmS name: It starts as daemon: Screen session in detached mode.
  • -D (-r): It detaches and logs out remote (and reattach here).
  • -D -RR: It does whatever is needed to get a screen session.
  • -e xy: It changes the command characters.
  • -f: It makes the flow control on, -fn = off, -fa = auto.
  • -h lines: It sets the size of the scrollback history buffer.
  • -i: It interrupts output sooner when flow control is on.
  • -l: It makes the login mode on (update /var/run/utmp), -ln = off.
  • -ls [match]: It displays all the attached screens.
  • -L: It turns on output logging.
  • -m: It ignores $STY variable, do create a new screen session.
  • -O: It chooses optimal output rather than exact vt100 emulation.
  • -p window: It preselects the named window if it exists.
  • -q: It quiets startup. Exits with non-zero return code if unsuccessful.
  • -Q: Its commands will send the response to the stdout of the querying process.
  • -r [session]: It reattaches to a detached screen process.
  • -R: It reattaches if possible, otherwise starts a new session.
  • -S sockname: It names this session .sockname instead of …
  • -t title: It sets title (window’s name).
  • -T term: It uses term as $TERM for windows, rather than “screen”.
  • -U: It tells screen to use UTF-8 encoding.
  • -v: It prints “Screen version 4.06.02 (GNU) 23-Oct-17”.
  • -x: It attaches to a not detached screen (Multi display mode).
  • -X: It executes as a screen command in the specified session.

Shortcut keys Options:

  • Ctrl-a + c: It creates a new window.
  • Ctrl-a + w: It displays the list of all the windows currently opened.
  • Ctrl-a + A: It renames the current window. The name will appear when you list the list of windows opened with Ctrl-a + w.
  • Ctrl-a + n: It goes to the next window.
  • Ctrl-a + p: It goes to the previous window.
  • Ctrl-a + Ctrl-a: It goes back to the last window used.
  • Ctrl-a + k: It closes the current window (kill).
  • Ctrl-a + S: It splits the current window horizontally. To switch between the windows, do Ctrl-a + Tab.
  • Ctrl-a + |: It splits the current window vertically.
  • Ctrl-a + X: Closes active split window.
  • Ctrl-a + Q: Closes all split windows.
  • Ctrl-a + d: It detaches a screen session without stopping it.
